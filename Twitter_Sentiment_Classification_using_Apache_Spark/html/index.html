Twitter Sentiment Classification using Apache Spark

In this assignment, we will be performing sentiment analysis on tweets using Apache Spark with different machine learning techniques. We will learn how to extract features from data and build supervised learning models to automatically classify tweets as either positive or negative (sentiment). Classifiers we will be examining are Naive Bayes, Logistic Regression.

1. Tweet processing steps:

  1. Removed new lines from beginning and end of the tweet text if present.
  2. Replaced new lines in between the text of the tweet with space.
  3. Replaced the whole word containing http/https or www. with the word URL
  4. Replaced @<username> with word AT_USER in the tweet text.
  5. Removed all the words which does not starts with an alphabet.
  6. Removed all kinds of punctuation marks present in the tweet text using built-in string.punctuation. This also includes ‘#’ in it so the #xyz words will be replaced by xyz.
  7. Replaced all words with more than two characters repeating with words in which characters are repeated exactly twice. (Example: exxxtenddddd to exxtendd)
  8. Split the tweet text on spaces.
  9. For each token got after splitting, apply Porter stemmer to get the exact English word of that token and store in a list.
  10. Got distinct tokens of the stemmed tokens.
  11. For each distinct stemmed token, if it is a stop word, removed it.

2. Feature space description:

The feature space follows bag of words model which consists of distinct words from the training.csv tweets. The tweets are cleaned and processed as described in the previous step. This list of tokens got from the processing of tweets is then converted to sparse vectors using HashingTF.transform api with feature size of 50000. Only unigrams are considered while processing the tweets.

3. Parameter tuning:

I tried varying the number of features passed to hashingTF. I started with 50000 (which I got while referring a website) I got the maximum accuracy for the classifiers. When tried for other numbers by trial and error method the accuracy decreased.

6. Evaluation metrics description:

Precision in this assignment context is how many positively classified tweets were actually positive. Recall in this context is how good a classifier classifies a positive tweet as positive.

F1 score in this context is the accuracy measure in terms of precision and recall

True positives in this case tells that how many actual positive tweets have been classified as positive.

True negatives in this case tells us how many actual negative tweets are predicted as negative tweets.

False positives in this case how many tweets inspite of being negative are predicted as positive.

False negatives in this case how many tweets inspite of being positives are predicted as negative.

8. Best performing classifier:

Although the test accuracy of both Logistic regression and Naive Baysian classifiers does not differ too much Naive Baysian classifier beats logistic regression in test accuracy. Reason may be that since the train data should be (refer: link ) much large for logistic regression to beat Naive bayes and since the data is just around 80000 tweets some more tweets would have made logistic regression a better classifier. Also the logistic regression uses a threshold comparing to which it classifies the data as positive or negative. But Naive Baysian classifier calculates probability of tweet having positive or negative sentiments and then whichever is greater classifies into that particular class. Also the logistic regression model tends to overfit the training model (which is obvious from the training accuracy) and hence Naive Baysian outperforms Logistic regression.
